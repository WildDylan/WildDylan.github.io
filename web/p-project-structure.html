项目目录结构

规律与整齐的生活是否会让你感到厌烦

有条不紊的做事情，我们先建立一个简单的目录结构，打开WebStorm：

  • 1. 点击Create New Project
  • 2. 左侧选择Empty Project, 并在右侧输入你想将项目创建的位置
  • 3. 点击创建

创建成功后截图

接下来，我们创建我们自己的工作目录，结合我们的功能，我们先宏观考虑一下怎么划分我们的项目结构（说的易懂一些）：

  • 1. 登录、注册、主页、详情共4个界面，这些界面的数据来源与操作都独立，划分为4个模块，每个模块拥有自己的资源文件，工具文件，数据等。
  • 2. 项目分为共享服务层（提供所有模块公共的服务），界面拥有相同内容。

显然，第2个貌似更好一点，第一种更适合于多个项目依赖的结构，互不影响。第二种更适合于平时我们的开发，所有公用的css、js、html文件扔到一起，然后每个模块再独立出去：

目录结构介绍：

  • 1. css 公用样式库的css文件夹
  • 2. js 公用js库的文件夹，比如说jQuery等等
  • 3. utils 公用的自定义的一些js工具类
  • 4. views 每一个视图我们称为一个模块
    • 1. components 公用组件，比如说我们会吧header, footer写到这里
    • 2. detail 详情模块
    • 3. signin, signup 登录注册模块
  • 5. index.html 入口界面
